Phase：爲 Hexo 和 Octopress 設計的華麗主題

2013年01月03日 Hexo, Octopress, Phase, 主題, 鏈接

非常漂亮的主題，又是 @tommy351 的作品，Phase 原本是 @tommy351 的私用主題，現在公開發佈，同時提供 Hexo 和 Octopress 的版本，主題以 Slash 為基礎開發（看看人家的設計，再看看我這個 _(:з」∠)_），最初爲 Octopress 設計，現在移植到了 Hexo ，Hexo 版還擁有更多的特性，作爲 Hexo 的作者往後的主題開發也都會以 Hexo 為主了（是不是可以成爲一個遷移到 Hexo 的理由？）。

就像作者所說，這個主題非常的絢麗，但也非常吃資源，可以打開預覽頁面感受下，感興趣的到作者發佈頁面查看詳細說明吧。

Hexo：基於 Node.js 的 Blog 框架

2013年01月03日 Hexo, 鏈接

來自寶島臺灣的 @tommy351 的作品，以前在 Google+ 上就使用過他寫的腳本 「Google+ Hover Zoom」，我現在所用的主題也是修改自他原創的 Slash 主題。

很早就在作者的 Blog 上看到這個，只是當時 Octopress 都沒弄好，現在看到 @lucifr 完全遷移到了 Hexo，就抓來本地試了下。

相比基於 Ruby 的 Jekyll 和 Octopress，使用 Node.js 驅動的 Hexo 網頁生成速度更快（得益於 Google 的 V8 引擎？），而且作者更新給力，現在已經可用，對我來說基本能替代 Octopress 了，不過完全遷移要花點時間（其實從 Octopress 遷移是最簡單的，Hexo 引入了很多 Octopress 的標籤外掛，文章都不用怎麼改，只是考慮要不要扔掉現在這渣主題），畢竟 Octopress 也花了好久才調教的差不多，而且現在文章也不多還感受不到 Octopress 的龜速，先放本地慢慢調教，說不好什麼時候真的就遷移到 Hexo 了。

使用 Obfsproxy 搭配 Shadowsocks 安全高效的訪問互聯網

2013年01月01日 obfsproxy, 代理, 鏈接

通過 obfsproxy 混淆 shadowsocks 流量，避免代理被X，其它代理也可以舉一反三在中間加上一層 obfsproxy 來混淆數據，整理下思路：

  1. 服務端通過 obfsproxy 混淆轉發代理服務，服務器防火牆只需打開 obfsproxy 的轉發端口即可；
  2. 客戶端用 obfsproxy 接收服務器相應端口的混淆數據並轉發到需要的端口（根據實際情況作爲代理客戶端的服務端口或直接作爲代理端口）使用。

在 Amazon EC2 免費主機上弄了個，用了一個星期，非常穩定，強力推介！
